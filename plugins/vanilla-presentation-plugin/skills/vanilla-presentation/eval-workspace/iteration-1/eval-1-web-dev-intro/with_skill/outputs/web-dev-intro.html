Section 01

HTML · CSS · JavaScript 기초

<!-- HTML: 구조 -->
<div class="card">
  <h1>Hello, Web!</h1>
  <p>반갑습니다.</p>
</div>

/* CSS: 스타일 */
.card {
  background: #1e1e2e;
  padding: 20px;
  border-radius: 8px;
}

// JavaScript: 동작
const btn = document
  .querySelector('button');
btn.addEventListener(
  'click', () => {
    alert('클릭!');
  });
HTML — 뼈대 태그(Tag)로 콘텐츠의 의미와 구조를 정의합니다. <h1>, <p>, <div>, <a> 등이 기본 빌딩 블록입니다.
CSS — 디자인 선택자(selector)와 속성(property)으로 색상·크기·레이아웃을 꾸밉니다. Flexbox와 Grid가 현대 레이아웃의 핵심입니다.
JavaScript — 동작 페이지를 살아있게 만드는 프로그래밍 언어입니다. 변수, 함수, 조건문, 반복문이 기본 개념입니다.
Section 02

DOM 조작

// 요소 선택
const title =
  document.getElementById('title');
const items =
  document.querySelectorAll('.item');

// 내용·스타일 변경
title.textContent = '새 제목';
title.style.color = '#E8C97A';

// 클래스 조작
title.classList.add('active');
title.classList.toggle('hidden');

// 새 요소 생성 & 삽입
const li = document.createElement('li');
li.textContent = '새 항목';
document.querySelector('ul')
  .appendChild(li);
DOM이란? 브라우저가 HTML을 파싱해 만드는 트리 구조입니다. JavaScript는 이 트리를 읽고 수정해 화면을 동적으로 바꿉니다.
핵심 API getElementById · querySelector로 요소를 선택하고, textContent · classList · setAttribute로 변경합니다.
가상 DOM vs 실제 DOM React 등 프레임워크는 가상 DOM으로 변경 최소화 → 성능 최적화. 원리를 이해하면 프레임워크도 쉬워집니다.
Section 03

이벤트 처리

// 기본 이벤트 리스너
const btn =
  document.querySelector('#btn');

btn.addEventListener('click', (e) => {
  console.log('클릭됨!', e.target);
});

// 이벤트 위임 (delegation)
document.querySelector('ul')
  .addEventListener('click', (e) => {
    if (e.target.matches('li')) {
      e.target.classList
        .toggle('selected');
    }
  });

// 키보드 이벤트
document.addEventListener(
  'keydown', (e) => {
    if (e.key === 'Enter')
      submitForm();
  });
주요 이벤트 종류 click · input · submit · keydown · mouseover · scroll · resize
이벤트 버블링 이벤트는 자식 → 부모로 전파됩니다. e.stopPropagation()으로 전파를 막고, e.preventDefault()로 기본 동작을 취소합니다.
이벤트 위임 패턴 동적으로 추가된 요소에도 이벤트를 적용하려면 부모에 리스너를 달고 e.target을 확인하세요. 메모리 효율도 높아집니다.
Section 04

비동기 프로그래밍

// Promise 기반 fetch
fetch('https://api.example.com/data')
  .then(res  => res.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

// async / await (더 읽기 쉬운 방식)
async function loadUser(id) {
  try {
    const res = await fetch(
      `/api/users/${id}`
    );
    if (!res.ok) throw new Error(res.status);
    const user = await res.json();
    renderUser(user);
  } catch (e) {
    showError(e.message);
  }
}

// 병렬 요청
const [posts, users] =
  await Promise.all([
    fetch('/posts').then(r=>r.json()),
    fetch('/users').then(r=>r.json()),
  ]);
왜 비동기가 필요한가? JavaScript는 싱글 스레드입니다. 네트워크 요청처럼 시간이 걸리는 작업을 동기로 처리하면 브라우저가 멈춥니다. 비동기로 처리해 UI를 유지합니다.
Promise → async/await Promise 체이닝보다 async/await가 더 직관적입니다. 에러는 반드시 try/catch로 처리하세요.
병렬 vs 순차 실행 독립적인 여러 요청은 Promise.all()로 병렬 실행해 전체 시간을 단축하세요.
